# 마이 펫 작업 기록

이 문서는 마이 펫 기능을 개발하면서 사용자가 요청한 내용, 판단 이유, 실제 수정 내용, 검증 결과를 한국어로 정리하는 기록이다.
앞으로 마이 펫 관련 코드를 수정할 때는 새 `2026-*.md` 파일을 만들지 않고 이 파일에 날짜별로 이어서 기록한다.

## 기능 기준

- 화면 명칭은 `마이 펫`으로 통일한다.
- 오른쪽 아래 `+` 버튼으로 펫 등록을 시작한다.
- 등록 화면 왼쪽 위에는 닫기 버튼이 아니라 뒤로가기 화살표를 사용한다.
- 등록 단계 이동 후에도 이전 단계로 돌아갈 수 있어야 한다.
- 필수 항목은 종류, 종, 성별, 이름이다.
- 사진, 체중, 생일, 입양일은 선택 항목이다.
- 생일과 입양일에는 `모르겠어요` 선택지를 두지 않고, 입력하지 않아도 다음 단계로 넘어갈 수 있게 한다.
- 체중은 g 또는 kg 단위를 선택할 수 있게 한다.
- 펫 등록 완료 후에는 필수 정보 저장을 먼저 끝내고, 사진·몸무게·생일·입양일 같은 추가 정보는 나중에 추가할 수 있게 한다.

## 등록 흐름

1. 종류 선택: 파충류, 조류, 설치류, 양서류, 기타
2. 종·성별 선택: 선택한 종류에 맞는 종 목록과 성별을 같은 단계에서 선택
3. 이름 입력: 닉네임 또는 이름 입력
4. 등록 완료: 필수 정보 저장 완료
5. 추가 정보: 사진, 몸무게, 생일, 입양일은 선택적으로 추가

수정 화면에서는 기존 등록값을 채운 상태로 모든 단계와 추가 정보 입력을 열어둔다.

## 목록 화면 기준

- 동물 이름, 종, 분류를 검색할 수 있어야 한다.
- 분류 필터는 전체, 파충류, 설치류, 양서류, 조류, 기타를 제공한다.
- `전체`를 제외한 분류 태그는 여러 개를 동시에 선택할 수 있다.
- `전체`를 누르면 선택한 분류 필터를 초기화한다.
- 선택된 태그는 초록색이 아니라 네이비 또는 청회색 계열로 표시한다.
- 등록된 펫 카드에는 사진이 있으면 사진을 보여주고, 사진이 없으면 분류별 기본 아이콘을 보여준다.
- 카드의 점 3개 메뉴에는 수정과 삭제를 넣는다.
- 삭제 시 확인 모달을 띄운다.
- 카드 자체를 누르면 다이어리 또는 관리 화면으로 이동할 수 있어야 한다.
- 점 3개 메뉴 클릭은 카드 클릭 이벤트와 충돌하지 않아야 한다.

## 데이터 연결 기준

- 펫 데이터는 기존 프로젝트의 실제 펫 데이터 구조와 저장 방식을 유지한다.
- 기록 화면에도 마이 펫 데이터를 전달한다.
- 오늘 관리 요약은 다이어리의 `daily_tasks` 데이터를 사용한다.
- 최근 기록 요약은 `care_records` 데이터를 사용한다.
- 오늘 계획이 없으면 진행률 막대를 억지로 표시하지 않는다.
