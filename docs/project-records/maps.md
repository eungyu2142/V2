# maps 작업 기록

지도/리뷰 영역의 기획, 구현, 오류 처리, UI 수정 내용을 날짜별로 기록한다.

## 기준

- 지도 화면은 전체 화면 지도를 기본으로 한다.
- 검색창과 동물 분류 태그는 지도 위 오버레이로 둔다.
- 네이버 지도 SDK는 지도 표시, 내 위치, 마커 표시만 담당한다.
- 네이버 지역 검색 API 키와 Secret은 프론트에 노출하지 않는다.
- 병원 검색은 Supabase Edge Function 또는 서버에서 수집한 데이터만 통해 처리한다.
- 병원 리뷰는 외부 사이트로 이동하지 않고 앱 내부에서 열린다.
- 병원 좋아요, 리뷰 좋아요, 임시저장, 내가 쓴 리뷰는 프로필의 나의 활동과 연결되어야 한다.
- 새 UI는 네이비/블루 디자인 시스템을 따른다.

## 2026-07-10 네이버 지도 초기 위치 표시

### 요청 요약
- 지도 첫 진입 시 내 위치 기준으로 열리고, 내 위치 마커가 바로 보여야 했다.
- 위치 안내 문구와 불필요한 오류 문구는 화면에서 줄여야 했다.

### 작업 내용
- 지도 진입 시 네이버 지도 SDK 로드와 브라우저 Geolocation 요청을 함께 처리하도록 정리했다.
- 위치 권한이 허용되면 현재 위치를 지도 중심 좌표로 사용하고 내 위치 마커를 표시하도록 했다.
- 위치 권한 실패 시에도 서울 기본 좌표로 지도 자체는 뜨게 fallback을 유지했다.
- 초기 위치 안내 문구를 제거하고 지도 중심 이동은 조용히 처리하도록 했다.

### 오류 처리
- 빌드 중 누락된 유틸 함수와 export 문제를 정리했다.
- 브라우저 자동 검증은 로컬 Chrome 권한 문제로 제한이 있었고, 빌드 검증으로 대체했다.

### 검증
- `npm run build` 통과.

## 2026-07-18 병원 검색 API 연결 및 마커 표시

### 요청 요약
- QNA 병원 첨부에서는 지역 검색 API 결과가 나오는데 지도에서는 병원이 보이지 않는 문제를 확인해야 했다.
- 특수동물병원, 이국동물병원, 파충류 동물병원, 조류 동물병원 등 검색어 기준으로 병원을 지도에 띄워야 했다.

### 작업 내용
- QNA 병원 첨부에서 사용하던 `search-hospitals` Edge Function 흐름을 지도 검색에도 연결했다.
- 일반 상점, 카페, 펫샵 등 병원이 아닌 결과가 섞이지 않도록 병원 후보 필터를 강화했다.
- 네이버 지역 검색 API 응답의 `mapx`, `mapy`를 지도용 위도/경도에 맞게 변환하는 로직을 보정했다.
- 지도 첫 진입 시 병원 검색이 한 번 실행되도록 해 병원 H 마커가 표시되게 했다.
- 병원 검색 결과가 없더라도 지도는 계속 표시되도록 분리했다.

### 오류 처리
- Edge Function 기본 검색어가 깨진 문자열로 남아 있던 문제를 수정했다.
- 검색 응답 좌표가 일반 위도/경도가 아니라 스케일링된 좌표로 들어오는 케이스를 확인하고 보정했다.

### 검증
- Edge Function 직접 호출 결과 `status: 200`, 병원 결과 반환 확인.
- `npm run build` 통과.

- 같은 검색 조건 반복 호출을 막아 불필요한 API 요청을 줄였다.
- 오른쪽 패널 태그가 줄바꿈되더라도 잘리지 않게 크기와 wrap을 보정했다.

## 2026-07-19 전국 특수동물 병원 데이터 수집 구조

### 요청 요약
- 앱이 열릴 때마다 네이버 지역 검색 API를 실시간 호출하지 말고, 서버에서 주 1회 수집한 데이터를 앱이 읽는 구조가 필요했다.
- 전국 시/군/구와 여러 특수동물 병원 키워드로 검색해 중복 제거한 JSON을 저장해야 했다.

### 작업 내용
- `scripts/collect-exotic-hospitals.mjs` 수집 스크립트 구조를 만들었다.
- 검색 키워드별 동물 분류를 `reptile`, `bird`, `rodent` 등으로 기록하도록 했다.
- 병원명과 주소를 정규화해 중복 병원을 병합하고, `matchedQueries`, `evidence`, `sources`를 남기도록 설계했다.
- 구체 분류가 없는 후보 병원은 앱에서 `기타`로 보이도록 정리했다.
- GitHub Actions 주간 수집 워크플로를 추가해 `public/data/exotic-hospitals.json` 갱신 구조를 마련했다.
- 지도와 QNA 병원 검색은 저장된 JSON을 우선 읽도록 방향을 잡았다.

### 오류 처리
- 기존 수집 스크립트의 문자열/JSON 구조가 깨져 있어 새 구조로 정리했다.
- 429 rate limit 방지를 위해 낮은 동시성, 요청 간 delay, backoff 재시도 정책을 계획에 포함했다.

### 검증
- 당시 수집 기능은 구조 정리 단계였고 실제 전체 수집은 추가 확인이 필요했다.

## 2026-07-19 병원 상세/리뷰 1차 개편

### 요청 요약
- 병원 정보와 리뷰가 지도 안에서 자연스럽게 열려야 했다.
- 리뷰 보기가 외부 병원 사이트로 이동하면 안 되고, 리뷰가 없으면 없다고 보여야 했다.
- 리뷰 작성 버튼을 통해 앱 내부 리뷰 작성이 가능해야 했다.

- 리뷰가 없는 병원은 상세에서만 빈 리뷰 안내가 나오게 하고, 병원 검색 목록에서는 빈 리뷰 문구를 제거했다.
- 리뷰 작성 폼을 `src/features/hospital-map/HospitalReviewForm.tsx`로 분리했다.
- 리뷰 작성 시 마이 펫, 방문일, 비용, 진단, 진료 내용, 처방/약 정보, 리뷰 태그, 본문을 저장하도록 확장했다.
- 리뷰 작성 후 해당 날짜의 다이어리 병원 기록과 연결되는 흐름을 추가했다.

- 병원 마커가 타원형이라 어색하므로 더 예쁜 형태가 필요했다.
- 리뷰 영역이 빽빽했고, 리뷰 작성 폼에서 닉네임, 동물 분류, 세부 종 입력이 불필요했다.
- 마이 펫을 선택하면 세부 종과 분류는 자동으로 들어가야 했다.
- 약봉투 인식은 유지하되 복용 안내와 복용량은 제거하고, 약 이름 대신 약 종류를 기록해야 했다.
- 기존 리뷰에서 `익명`이 보이는 문제를 없애야 했다.

### 작업 내용
- 모바일 동물 분류 태그 칩의 높이, 폰트, 아이콘, gap을 줄이고 가로 스크롤을 안정화했다.
- 태그 클릭 시 `setSheetDismissed(false)`가 실행되지 않도록 하여 바텀시트가 자동으로 열리지 않게 했다.
- 병원 마커를 기존 타원형 H 버튼에서 파란 지도 핀 형태로 바꿨다.
- 선택된 병원 마커는 진한 네이비로 강조하고, 신뢰도 배지는 작은 점으로만 표시했다.
- 리뷰 요약, 평점 분포, 리뷰 카드, 태그의 padding과 gap을 정리해 밀도를 낮췄다.
- 리뷰 작성 폼을 마이 펫 선택, 진료 정보, 약봉투 사진, 리뷰 태그, 리뷰 본문 구조로 단순화했다.
- 닉네임 입력을 제거하고 프로필 닉네임 또는 사용자명을 리뷰 작성자로 사용하도록 했다.
- 마이 펫 선택값에서 동물 분류와 세부 종을 자동으로 리뷰 데이터에 반영하도록 했다.
- 약봉투 인식은 `recognize-medication-bag` Supabase Edge Function 호출 구조를 유지했다.
- 약봉투 폼에서는 약 종류, 복용 시작일, 종료일, 하루 복용 횟수만 남겼다.
- 다이어리 약 복용 계획 연결은 약 종류만 있어도 생성될 수 있도록 `dose` 필드를 선택값으로 낮췄다.
