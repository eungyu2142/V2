# UI 작업 기록

## 역할

색상, 레이아웃, 컴포넌트 모양, 아이콘, 버튼, 카드, 지도 오버레이, 하단 내비게이션 등 시각 UI 변경 사항을 기록한다.

## 기준

- 앱 UI는 민트/아이보리 기반의 깔끔한 스타일을 유지한다.
- 네이버 지도 UX는 참고하되, 앱 자체 UI 정체성은 특수동물 케어 PWA에 맞춘다.
- 화면 요소가 웹 브라우저와 모바일 화면에서 깨지지 않아야 한다.

## 2026-07-22 레이아웃 시스템 정리

## 요청 요약

현재 페이지의 컴포넌트 크기와 정렬이 제각각이라 화면이 지저분하게 보이는 문제를 해결하기 위해, 앱 전체에 일관된 컨테이너·그리드·카드·간격 기준을 적용해 달라는 요청이었다.

## 분석·판단 이유

기존 CSS에는 페이지별 오버라이드가 많이 누적되어 같은 카드와 패널도 서로 다른 폭, gap, radius, 높이를 사용하고 있었다. 따라서 개별 컴포넌트를 하나씩 수정하기보다, 지도 화면을 제외한 일반 페이지에 공통 컨테이너와 반응형 Grid 기준을 최종 우선순위로 적용하는 방식이 가장 안정적이라고 판단했다. 지도는 전체 화면 지도가 핵심이므로 max-width 컨테이너 대상에서 제외했다.

## 핵심 변경 내용

- 일반 페이지의 `.app-main`과 `.top-bar`를 최대 1280px 컨테이너로 중앙 정렬했다.
- 마이펫, QNA, 나눔, 프로필 화면에 공통 gap, 카드 radius, 패널 padding, 카드 shadow 기준을 적용했다.
- 카드 목록은 모바일 1열, 태블릿 2열, 데스크톱 3열, 넓은 화면 일부 4열 Grid로 정리했다.
- 카드 내부는 `flex-direction: column`과 `margin-top: auto`를 사용해 하단 버튼 영역이 같은 축에 정렬되도록 맞췄다.
- 다이어리 화면은 최대 1280px 컨테이너 안에서 데스크톱 2열, 태블릿/모바일 1열로 정리했다.
- 버튼, 태그, 카드, 패널은 푸른빛 네이비 디자인 시스템의 border/radius/gap 기준을 따르도록 맞췄다.

## 남은 작업

- 실제 브라우저에서 각 탭별 화면을 보면서 특정 카드 내부 콘텐츠가 너무 길 때의 줄임 처리와 모바일 스크롤 체감을 추가로 점검해야 한다.
- `App.css`에 과거 오버라이드가 많이 누적되어 있으므로, 이후 리팩터링 단계에서는 공통 토큰과 페이지별 CSS를 분리하는 정리가 필요하다.
