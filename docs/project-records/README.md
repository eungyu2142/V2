# 프로젝트 작업 기록

이 폴더는 기능 구현 코드가 아니라, 사용자가 입력한 프롬프트를 수행하면서 각 기능에서 완료한 내용, 문제 인식, 해결 과정을 기록하기 위한 공간이다.

기능 구현 기준은 항상 `docs/APP_REQUIREMENTS.md`를 우선한다. 사용자의 최신 요구사항이 기존 구현 또는 기존 기록과 충돌하면 최신 요구사항을 우선한다.

## 기록 원칙

- 사용자가 한국어로 강조한 요구사항은 한국어 그대로 기록한다.
- 각 작업은 관련 기능 파일에 `문제 인식`, `해결 과정`, `완료 기록`으로 남긴다.
- 여러 기능에 영향을 주는 작업은 관련 기능 파일과 `all.md`에 함께 기록한다.
- UI/UX, 내비게이션, 반응형 화면, 모바일/PWA 화면 깨짐 여부는 별도로 확인하고 기록한다.
- 기록 파일은 앱 기능 코드로 사용하지 않는다.

## 최초 사용자 지시 기록

### 문제 인식

사용자는 앞으로 프롬프트를 수행할 때마다 기능별로 완료한 내용을 누락 없이 기록할 수 있는 별도 프로젝트 기록 파일 체계를 요청했다. 이 기록 파일들은 코딩 기능을 담당하지 않고, 작업 결과와 문제 인식 및 해결 과정을 저장하는 목적이다.

### 해결 과정

기능별 기록 파일을 `docs/project-records` 아래에 분리해서 만들고, 전체 화면 및 PWA 반응형 요구사항을 `all.md`에 별도로 둔다. 또한 파란색 박스처럼 화살표 흐름을 볼 수 있는 전체 연결 파일로 `NAVIGATION_FLOW.md`를 만든다.
