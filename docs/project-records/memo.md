## 역할

기록, 캘린더, 날짜별 메모, 먹이, 무게, 탈피, 배변, 청소, 병원, 기타 기록 관련 작업을 기록한다.

## 기준

- 리뷰 작성 후 해당 날짜의 동물 기록에 연결되어야 한다.
- 질문 작성 후 동물 기록에 연결되어야 한다.
- 임시저장 데이터는 프로필의 나의 활동과 연결되어야 한다.

## 2026-07-09 기록 탭 복구

- 앱에 등록된 실제 펫 목록을 기록 탭과 연결했다.
- 더미 데이터 없이 월간 달력 셀에 기록 종류를 표시한다.
- 미래 기록 작성 제한은 유지하고 미래 예정 알람은 표시한다.
- 기록 작성은 종류, 상세, 메모, 사진을 한 화면씩 진행하며 화살표로 이전 단계로 돌아간다.
- 먹이는 선택과 직접 입력, 탈피 등은 종류별 세부 선택을 지원한다.
- 사진 입력은 기본 파일 버튼 대신 민트색 선택 UI를 사용한다.
- 펫 미등록 상태에서 `+`를 누르면 펫 등록 안내와 이동하기 버튼을 표시한다.
- 종류 선택 창이 열리면 플로팅 `+` 버튼을 숨긴다.
- `예정 알람`은 별도 전체 화면에서 제목, 종류, 반복/1회, 일정, 메모를 단계별로 작성하며 저장할 때만 목록에 추가한다.
- 기록과 알람은 `localStorage`에 저장한다.

### 오류 기록

- 처리: 권한 허용 방식으로 동일 빌드를 재실행했다.
- 확인: `npm run build` 최종 통과.

## 2026-07-11 알람 실행부 추가

- 문제: 예정 알람을 저장하고 달력에 표시할 수는 있었지만, 지정 시간에 실제 브라우저 알림을 띄우는 실행 로직이 없었다.
- 원인: feeding_reminders 데이터는 저장/조회만 되고 Notification API, 권한 요청, 시간 체크 타이머와 연결되어 있지 않았다.
- 처리: 기록 화면 우측 예정 알람 영역에 알림 켜기 버튼을 추가하고, 권한 허용 후 앱이 켜져 있을 때 15초마다 현재 시간과 알람 시간을 비교해 브라우저 알림을 띄우도록 구현했다.
- 처리: 같은 분에 같은 알람이 반복으로 여러 번 울리지 않도록 fired reminder key를 localStorage에 저장한다.
- 제한: 웹앱 특성상 앱이 완전히 종료된 상태의 강제 벨소리/백그라운드 알람은 보장하지 않는다.
- 변경 파일: src/features/diary/DiaryPage.tsx, src/features/diary/DiaryPage.css
- 오류 기록: TypeScript DOM 타입에 NotificationOptions.renotify가 없어 빌드 오류가 발생했고, 해당 옵션을 제거했다.
- 검증: tsc -b, npm run build 통과.

## 분석·판단

그래서 컴포넌트 구조를 바꾸지 않고, 모바일 전용 최종 CSS override를 추가해 `플랜` 탭에서는 캘린더 영역과 선택 날짜 상태 영역을 확실히 숨기도록 처리했다.

- 캘린더 아래에 붙는 `.selected-date-status`도 플랜 탭에서는 함께 숨기도록 했다.

### 분석·판단

`DiaryPage.css`에서 탭 높이를 고정했지만, 전역 스타일인 `src/App.css`의 `button.active` 규칙이 `.diary-mobile-tabs button.active`에도 적용되고 있었다. 이 전역 규칙은 active 버튼에 그림자를 강제로 넣기 때문에, 모바일 탭을 누를 때 버튼이 커지거나 높이가 바뀌는 것처럼 보일 수 있었다.

또한 브라우저 기본 버튼 스타일이나 텍스트 줄바꿈이 개입하지 않도록 다이어리 탭 버튼의 박스 모델을 더 강하게 고정했다.

- `src/App.css`
- `src/features/diary/DiaryPage.css`

- 버튼에 `white-space: nowrap`, `appearance: none`, `box-sizing: border-box`를 적용해 텍스트 줄바꿈이나 기본 버튼 스타일로 인한 높이 변화를 방지했다.

### 요청 요약

### 분석·판단

두 가지 원인이 남아 있었다.

### 수정 파일

- `src/App.css`
- `src/features/diary/DiaryPage.css`

- 다이어리 모바일 탭에는 전역 button active의 `transform`과 `box-shadow`가 적용되지 않도록 `transform: none`, `box-shadow: none`을 강제했다.

## 추가 요청: 플랜/캘린더 상단 위치 통일

### 요청 요약

모바일 플랜 화면에서 `플랜 / 캘린더` 버튼과 동물 마크가 캘린더 화면을 눌렀을 때와 같은 높이에 있어야 한다는 요청. 상단 위치를 맞추면 아래 카드들은 자연스럽게 위로 올라와야 한다는 의도.

### 분석·판단

플랜과 캘린더는 같은 JSX 상단 구조를 공유하지만, CSS의 모바일 적용 범위가 일부는 `640px 이하`, 일부는 `900px 이하`로 나뉘어 있었다. 이 때문에 실제 모바일/태블릿 프리뷰 폭에 따라 펫 정보와 탭의 간격이 다르게 보일 수 있었다.

## 2026-07-22 다이어리 달력 클릭과 기록 라벨 개선

### 요청 요약

달력 날짜를 클릭했을 때 첫 클릭은 현재처럼 날짜 선택만 유지하고, 이미 선택된 같은 날짜를 한 번 더 클릭했을 때 세부 내용 화면으로 이동하게 해달라는 요청이었다. 또한 달력이 너무 작고 날짜 칸에 점만 표시되면 어떤 일을 했는지 알기 어렵기 때문에, 먹이, 물, 청소 같은 짧은 이름이 달력 안에 보이게 해달라고 했다.

### 분석·판단 이유

기존 달력은 날짜 선택과 상세 열기 동작이 한 번의 클릭에 섞일 수 있어 사용자가 단순히 날짜만 보고 싶을 때 화면이 갑자기 바뀔 가능성이 있었다. 그래서 선택과 상세 보기를 분리해 같은 날짜를 재클릭할 때만 상세 화면으로 넘어가게 했다.

이전 점 표시 방식은 깔끔하지만 사용자가 기록 종류를 바로 읽기 어려웠다. 이번 요청에서는 정보 확인성이 더 중요하므로 긴 문장 대신 먹이, 물, 청소, 배변, 약 같은 짧은 라벨형 배지로 바꾸고 모바일에서도 칸 높이를 키웠다.

- 캘린더 셀의 점 표시를 짧은 라벨 배지로 바꿔 먹이, 물, 청소 등 수행 항목을 바로 볼 수 있게 했다.
