# 지속 작업 기록 규칙

## 목적

사용자가 프롬프트로 요청한 기능과 UI 변경을 구현할 때마다 한국어 메모를 남겨, 이후 프로젝트 구조와 구현 과정을 학습할 수 있도록 한다.

## 기록 방법

- 기능별 문서가 있으면 해당 문서에 기록한다.
- 기능별 문서가 없으면 이 폴더에 새 문서를 만든다.
- 각 기록에는 요청 요약, 변경 이유, 수정 파일, 주요 동작 흐름, 검증 결과를 포함한다.
- 코드 변경과 문서 기록을 같은 작업 단위로 처리한다.

## 현재 적용

- QNA, 다이어리, 지도, 나눔 작업을 구현할 때 관련 문서에 한국어로 누적 기록한다.
- 빌드나 린트 오류가 발생하면 원인과 처리 결과도 함께 기록한다.
