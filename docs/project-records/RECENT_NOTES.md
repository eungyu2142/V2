# 최근 작업 기록

이 파일은 사용자가 프롬프트를 입력하고 Codex가 코드를 변경할 때, 그 과정을 공부할 수 있도록 한국어로 남기는 작업 일지다.

기존 `docs/project-records` 파일 중 일부는 예전 인코딩 문제로 글자가 깨져 보이는 구간이 있다. 기존 기록을 잃지 않기 위해 당장은 덮어쓰지 않고, 새 기록은 이 파일에 정상 한국어로 누적한다. 이후 기능별 문서를 정리할 때는 이 내용을 기준으로 옮기면 된다.

## 기록 운영 원칙

### 목적

단순히 “완료했다”만 남기지 않고, 다음 내용을 같이 기록한다.

- 사용자가 어떤 문제나 요구사항을 말했는지
- 기존 코드에서 어떤 문제가 있었는지
- 어떤 파일을 왜 바꿨는지
- DB, RLS, RPC, 인증 구조가 어떻게 바뀌었는지
- 검증을 어떻게 했고 결과가 어땠는지

### 앞으로의 방식

- 코드, DB, UI, 정책, 마이그레이션을 변경할 때마다 한국어 기록을 추가한다.
- 기록은 가능하면 `문제 인식`, `해결 과정`, `완료 기록`, `검증` 순서로 남긴다.
- 계정/로그인 작업은 account/login 범주로 기록한다.
- 프로필, 나의 활동, 로그아웃, 계정 삭제 작업은 profile 범주로 기록한다.
- 마이 펫 작업은 mypet, 기록/캘린더 작업은 memo, 지도/리뷰 작업은 maps, QNA는 QNA, 나눔은 share, 공통 UI/UX는 UI 또는 UX 범주로 기록한다.
- 여러 기능에 동시에 영향을 주는 작업은 공통 기록으로 함께 남긴다.

## 2026-07-22 아이디 찾기와 비밀번호 재설정

### 문제 인식

현재 앱은 사용자에게 이메일을 요구하지 않고 `아이디(username) + 비밀번호`로 로그인한다. 내부적으로는 Supabase Auth의 email/password 인증을 사용하기 때문에 `username@exopet.local` 형태의 가상 이메일을 생성한다.

사용자는 이메일을 모르기 때문에 일반적인 이메일 기반 비밀번호 찾기 화면을 그대로 사용할 수 없다. 그래서 닉네임과 등록한 반려동물 이름을 이용해 아이디를 찾고, 아이디와 등록한 반려동물 이름을 이용해 비밀번호를 재설정하는 흐름이 필요했다.

### 해결 과정

- `src/lib/auth.ts`에 아이디 찾기와 비밀번호 재설정용 클라이언트 함수를 추가했다.
- `src/components/AuthScreen.tsx`에 `아이디 찾기`, `비밀번호 재설정` 화면 모드를 추가했다.
- 아이디 찾기 화면은 `닉네임`, `반려동물 이름`을 입력받고, 성공하면 username만 표시한다.
- 비밀번호 재설정 화면은 `아이디`, `반려동물 이름`, `새 비밀번호`, `새 비밀번호 확인`을 입력받는다.
- Supabase 마이그레이션 `202607160002_account_recovery_by_pet.sql`을 추가했다.
- RPC `find_username_by_pet`은 `profiles.nickname`과 `pets.name`이 정확히 일치할 때만 username을 반환한다.
- RPC `reset_password_by_pet`은 username과 pet name이 정확히 일치할 때만 `auth.users.encrypted_password`를 bcrypt 해시로 갱신한다.
- 비밀번호는 클라이언트나 RPC 반환값으로 절대 반환하지 않는다.

### 완료 기록

- 로그인 화면에 `아이디 찾기`, `비밀번호 재설정` 버튼을 추가했다.
- 기존 로그인/회원가입 흐름은 유지했다.
- 아이디 찾기와 비밀번호 재설정은 이메일 입력 없이 동작한다.
- 잘못된 반려동물 이름을 입력하면 아이디 반환과 비밀번호 변경이 모두 거부된다.

### 검증

- `npm run build` 성공.
- `npm run lint` 성공.
- Supabase 원격 마이그레이션 반영 완료.
- 임시 계정으로 회원가입, 펫 등록, 아이디 찾기, 잘못된 펫 이름 거부, 비밀번호 재설정, 새 비밀번호 로그인, 테스트 계정 삭제까지 확인했다.

## 2026-07-22 로그아웃과 계정 삭제

### 문제 인식

사용자는 프로필에서 `로그아웃`과 `계정 삭제`를 사용할 수 있게 만들고 싶어 했다. 로그아웃은 이미 Supabase Auth 세션을 종료하는 흐름이 있었지만, 계정 삭제는 사용자가 실수로 누르면 되돌릴 수 없기 때문에 별도의 확인 절차가 필요했다.

### 해결 과정

- 기존 Supabase RPC `delete_own_account()`를 사용했다.
- 이 RPC는 로그인한 사용자의 `auth.uid()` 기준으로 `auth.users`에서 본인 계정을 삭제한다.
- `auth.users`와 연결된 `profiles`, `pets`, `care_records` 등 사용자 데이터는 외래키 `on delete cascade` 정책에 따라 함께 정리된다.
- `src/App.tsx`에 `deleteAccount` 함수를 추가했다.
- 계정 삭제 성공 후 `supabase.auth.signOut()`을 호출해 로컬 세션도 정리하도록 했다.
- 프로필 메뉴에 `계정 삭제` 항목을 추가했다.
- 계정 삭제 화면에서는 안내 문구를 보여주고, 입력창에 정확히 `계정 삭제`를 입력해야 삭제 버튼이 활성화되도록 했다.
- `src/App.css`에 위험 버튼과 안내 박스 스타일을 추가했다.

### 완료 기록

- 프로필 메뉴에서 `로그아웃`과 `계정 삭제`를 사용할 수 있다.
- 계정 삭제는 확인 문구 입력 없이는 실행되지 않는다.
- 삭제 실행 후 계정 삭제 RPC를 호출하고 자동 로그아웃한다.

### 검증

- `npm run build` 성공.
- `npm run lint` 성공.
- 빌드 중 chunk size 경고가 있었지만 실패가 아니며 기능 동작에는 영향이 없다.
