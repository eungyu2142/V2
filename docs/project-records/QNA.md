# QNA 작업 기록

## 역할

질문 목록, 질문 상세, 질문 작성, 댓글, 임시저장, 동물 기록 연결 관련 작업을 기록한다.

## 기준

- 질문 목록에서는 머리글이 보여야 한다.
- 질문 작성 후 동물 기록에 추가되어야 한다.
- 질문 임시저장은 프로필의 나의 활동과 연결되어야 한다.

## 2026-07-09 복구 기록

- 프론트엔드 명칭과 내부 탭 식별자를 `QNA`/`qna`로 고정했다.
- 상시 노출 입력창에서 제목, 내용, 작성자, 동물 종을 찾을 수 있다.
- 카테고리는 전체, 정보, 동물 병원, 건강/증상만 사용한다.
- 목록은 제목, 한 줄 미리보기, 작성자, 동물 종, 좋아요, 댓글, 작성 시간과 선택 사진을 표시한다.
- 상세 화면에서 좋아요 토글, 댓글 작성, 본인 댓글 삭제가 가능하다.
- QNA 작성 전 마이 펫 동물을 반드시 선택하며 카테고리, 제목과 사진, 내용과 동물 종류 태그 순서로 작성한다.
- 작성 화면은 왼쪽 위 화살표로 이전 단계 또는 QNA 목록으로 돌아간다.
- `scripts/guard-qna-label.mjs`를 빌드 전에 실행해 금지된 이전 명칭이 프론트엔드에 다시 들어오지 않게 했다.

## 2026-07-09 작성 흐름 수정

- 마이 펫 등록 여부와 관계없이 QNA를 작성할 수 있도록 변경했다.
- 마이 펫 선택은 선택 사항이며, 선택하면 이름 대신 동물 분류와 동물 종만 글에 연결한다.
- 마이 펫을 선택하지 않은 경우 동물 분류와 동물 종을 직접 선택하거나 입력할 수 있고, 두 항목 모두 선택 사항이다.
- 작성 단계는 카테고리, 동물 정보, 본문 작성의 3단계로 단순화했다.
- 제목, 내용, 사진 첨부는 마지막 한 화면에서 함께 작성한다.
- 목록과 상세 화면에는 동물 이름 대신 `동물 분류 · 동물 종`을 표시한다.

### 동물 없음 선택

- 동물 정보 단계에서 등록된 마이 펫 또는 `동물 없음`을 반드시 선택한다.
- `동물 없음`을 선택하면 동물 정보를 추가로 입력하지 않고 다음 단계로 이동한다.
- 해당 글은 목록과 상세 화면에서 `동물 없음`으로 표시한다.

# QNA 서비스형 질문 피드 개편

## 요청 요약

- 검색, 상태, 카테고리, 정렬이 흩어진 기존 QNA 목록을 질문 중심 피드로 전면 개선한다.
- 기존 Supabase 게시글, 상세, 댓글, 기록 첨부, 사진 첨부, 삭제 기능은 유지한다.

## 분석 및 판단

- 기존 QNA가 해결되지 않은 질문만 보여 주는 정렬 로직과 카드 직접 삭제 버튼을 사용하고 있어, 전체 데이터를 독립 상태 필터와 카테고리 필터로 조합하도록 변경했다.
- 카드 정보는 제목과 본문 요약을 먼저 보여 주고 동물, 작성자, 시간, 댓글 수, 첨부 정보를 뒤에 배치했다.

## 수정 파일

- `src/App.tsx`: QNA 필터 상태, 정렬, 카드 상태 계산, 검색 지우기, 소유자 메뉴, 접근성 키보드 동작
- `src/App.css`: QNA 웹 2열/모바일 1열, 검색창, 필터 행, 카드 및 첨부 요약 스타일

## 핵심 변경 내용

- 상태: `전체` / `답변 대기` / `답변 있음` / `해결 완료`
- 카테고리: `전체 카테고리` / `건강/증상` / `사육/관리`
- 정렬: `최신순` / `댓글 많은 순` / `오래된 순`
- 게시글 삭제는 본인 카드의 점 3개 메뉴에서만 노출
- 제목·본문 우선 카드와 기록·사진 첨부 요약 표시
- 웹 최대 2열, 모바일 1열, 검색 결과/필터 결과 빈 상태 처리

- QNA 목록 섹션의 테두리, 배경, 그림자, 내부 여백 제거
- QNA 게시글은 구분선만 유지하고 카드 배경과 그림자 제거
- 검색창은 하나의 외곽 박스만 유지
- 검색 input의 테두리, 배경, 그림자, 별도 radius 제거

# QNA 검색창 한 줄 배치 수정

## 요청 요약

- 검색 아이콘과 예시 placeholder가 검색창 안에서 한 줄에 자연스럽게 표시되어야 한다.

## 분석 및 판단

전역 `label` 스타일의 grid와 gap이 QNA 검색창에도 적용되어 아이콘과 input이 서로 다른 줄로 배치되고 있었다. QNA 검색 label에만 검색 아이콘 영역과 input 영역을 가진 2열 grid를 명시했다.

## 수정 파일

- `src/App.css`

## 핵심 변경 내용

- QNA 검색 label의 display, grid-template-columns, gap, align-items를 우선 적용
- 검색 아이콘과 검색 input을 같은 행에 배치

## 검증 결과

- 초기 빌드에서 다이어리의 사용되지 않는 `calendarItems` 변수 오류를 확인했다.
- 해당 사용되지 않는 변수도 제거한 뒤 빌드를 다시 확인한다.
- 이후 빌드에서 지도 리뷰 영역의 기존 TypeScript 오류가 다수 발견되어 전체 빌드는 실패했다.
- 검색창 수정과 직접 관련된 오류는 확인되지 않았다.

## 남은 작업

- 지도 리뷰 영역의 기존 TypeScript 오류를 별도 작업으로 정리해야 한다.

## 2026-07-19 블루 테마 정리

- QNA 상태·카테고리 컨트롤의 남은 민트/초록 색상을 블루·네이비 팔레트로 변경했다.
- 선택 상태는 진한 네이비, 기본 강조는 블루, 보조 배경은 연한 블루로 통일했다.
- 질문 작성 헤더 아래의 단계 키워드를 `질문`으로 정리했다.
- 버튼, 칩, 태그, 입력창, 카드, 바텀시트, 모달, 아이콘 버튼의 공통 디자인 레이어를 적용했다.
- QNA 저장과 상태 변경은 먼저 로컬 UI를 갱신한 뒤 Supabase 동기화를 시도하도록 정리했다.
- 검증: `npm run build` 통과.

## 2026-07-20 네이비 디자인 규칙

- QNA를 포함한 새 컴포넌트와 수정 컴포넌트에 블루 네이비 디자인 시스템을 적용하는 프로젝트 규칙을 추가했다.
- 기본 강조색은 `#1d4ed8`, 선택·활성 상태는 `#0f2742`, 보조 배경은 `#dbeafe`로 고정했다.
- 새 QNA UI에 민트·초록 계열을 추가하지 않도록 기준을 명확히 했다.

## 2026-07-22 QNA 기록 첨부 시각화 흐름

- 작성 화면에서 펫을 먼저 선택하면 해당 펫의 기록만 대상으로 사용하도록 했다.
- `동물 X`를 선택하면 기록 첨부 영역을 숨긴다.
- 기록이 없는 펫은 기록 첨부를 사용할 수 없게 처리했다.
- 기록 첨부 시 날짜를 별도로 선택하지 않고 선택한 펫의 최신 기록을 자동 연결한다.
- 작성 화면과 게시글에는 날짜·기록 종류·내용을 직접 노출하지 않고 `○○의 기록` 버튼만 표시한다.
- `○○의 기록` 버튼을 누르면 해당 펫의 전체 기록과 기록 유형별 개수를 그래프로 확인할 수 있다.
- 게시글 작성자 본인은 다이어리에서 다른 펫으로 전환할 수 있고, 다른 사용자는 기록을 읽기만 할 수 있게 했다.
- 다른 사용자는 기록 추가, 펫 전환, 알람 전환, 기록 삭제를 할 수 없도록 읽기 전용 흐름을 적용했다.

## 2026-07-22 QNA 피드와 검색 UI 정리

- 중복 QNA 제목을 제거하고 질문 목록을 평면 피드 구조로 정리했다.
- 박스 안의 카드 구조, 목록 외곽 박스, 게시글 카드 그림자를 제거하고 구분선만 유지했다.
- 검색창의 이중 테두리를 제거하고 아이콘과 검색 예시를 한 줄에 배치했다.
- 검색 예시는 `예: 먹이를 안 먹어요, 크레스티드 게코`로 변경했다.
- `전체 질문 보기` 버튼을 카테고리 태그와 같은 가로 영역에 배치했다.
- 카테고리는 `건강/증상`, `사육/관리`로 통일했다.
- 검증 중 지도 리뷰 영역의 기존 TypeScript 오류가 확인되어 QNA UI 수정과 분리해 남겼다.
