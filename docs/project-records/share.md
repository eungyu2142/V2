# share 작업 기록

## 2026-07-16 나눔 제목/지역/수정 개선

- 나눔 작성의 `머리글` 라벨을 `제목`으로 변경했다.
- 지역 입력은 주소나 지역을 직접 입력할 수 있게 하고, 서울/경기/인천 등 빠른 지역 선택 버튼을 추가했다.
- 이미 등록된 나눔 글도 카드와 상세 화면에서 `수정` 버튼으로 다시 작성 화면에 들어가 수정할 수 있게 했다.
- 수정 저장 시 기존 글 id를 유지해 새 글이 아니라 기존 글을 덮어쓴다.
- 카드 내부의 수정/완료/삭제 버튼이 상세 열기 클릭과 섞이지 않도록 이벤트 전파를 정리했다.
- 수정 파일: `src/App.tsx`, `src/App.css`.
- 검증: `tsc -b`, `npm run build` 통과.

## 2026-07-13 나눔 완료 상태 분리

- 나눔/무료분양 글에 `active`/`completed` 상태를 추가했다.
- 기본 나눔 목록은 `진행중` 글만 보여준다.
- 카드의 `나눔 완료` 버튼을 누르면 글이 완료 처리되어 기본 목록에서 사라진다.
- 상단 `진행중/완료` 필터를 눌러 완료된 글만 따로 볼 수 있게 했다.
- 완료 보기에서는 `완료` 태그와 `다시 나눔중` 버튼을 제공해 되돌릴 수 있다.
- 기존 글은 상태가 없으면 `진행중`으로 간주해 기존 데이터가 사라지지 않게 했다.
- 오류 대응: 작업 중 `src/App.tsx`와 이 문서에 mojibake가 섞여 GitHub Desktop 내장 Git의 `git show HEAD:...`로 정상 UTF-8 파일을 복구한 뒤 변경 사항을 다시 적용했다.
- 수정 파일: `src/App.tsx`, `src/App.css`, `docs/project-records/share.md`.
- 검증: `tsc -b`, `npm run build` 통과.

## 2026-07-10 나눔 작성 화면 표시 복구

- 나눔 작성 화면에서 첫 단계가 빈 화면으로 보이는 문제를 수정했다.
- 원인: `src/App.css`의 `.share-type-picker`, `.share-create-fields`가 `display: none`으로 남아 작성 단계 내용이 숨겨져 있었다.
- 조치: 작성 타입 선택 영역과 머리글/지역 입력 영역을 `display: grid`로 복구하고, 동물 선택 영역도 명시적으로 grid 레이아웃을 적용했다.
- 수정 파일: `src/App.css`.
- 검증: `tsc -b`, `npm run build` 통과.

## 2026-07-10 나눔 필터 탭 UI 축소

- 나눔 목록 필터가 큰 영역으로 펼쳐지지 않도록, 필터 버튼 아래에 작은 탭 패널만 열리게 변경했다.
- `종류`, `최신순/인기순`, `성별` 버튼은 선택값을 직접 표시한다.
  - 예: `파충류` 선택 시 버튼 라벨이 `파충류`로 바뀐다.
  - `전체`를 선택하면 버튼 라벨이 다시 `종류` 또는 `성별`로 돌아간다.
- 선택된 필터 버튼과 탭은 진한 민트 글씨/굵은 글씨로 표시한다.
- 모바일에서 세부 탭이 화면을 밀어내지 않도록 패널 내부를 가로 스크롤 구조로 정리했다.
- 수정 파일: `src/App.tsx`, `src/App.css`.
- 오류 대응: 초기화 과정에서 작성 화면의 `onSaveDraft`/`saveDraft` 연결이 빠져 TypeScript 오류가 발생해 임시저장 저장 함수와 버튼 문구를 복구했다.
- 검증: `tsc -b`, `npm run build` 통과.

## 2026-07-09 작성 시작 단계 단순화

- 작성 첫 화면은 세부 카테고리를 바로 보여주지 않고 `동물`과 `물품`만 선택하게 변경했다.
- 동물을 선택하면 다음 단계에서 `마이 펫` 또는 `다른 종 직접 입력`이 표시된다.
- 물품을 선택하면 다음 단계에서 `먹이/용품`과 해당 세부 카테고리가 표시된다.
- 기존의 머리글, 지역, 사진, 설명 단계는 선택 이후에 이어지도록 유지했다.

## 2026-07-09 GitHub 초기화 후 나눔 탭 복구

- 나눔을 동물 무료분양과 먹이/용품을 함께 다루는 `무료분양/나눔`으로 복구했다.
- 고정 검색창과 `종류`, `최신순/인기순`, `성별`, 전체 필터 버튼을 구현했다.
- 사진 중심 반응형 카드, 좋아요, 지역 및 분류/성별 표시를 연결했다.
- 작성은 상위/하위 카테고리 선택 후 동물은 마이 펫 또는 직접 종 입력, 먹이/용품은 물품 카테고리로 진행한다.
- 사진 파일은 `FileReader`로 읽어 파일명 대신 미리보기와 카드 이미지로 표시한다.
- 수정 파일: `src/App.tsx`, `src/App.css`.
- 오류: 문서 인코딩이 깨져 보이는 환경에서 첫 대형 패치 문맥 탐색이 실패해 함수/타입 경계의 작은 패치로 나눠 처리했다.
- 검증: TypeScript 검사와 Vite 프로덕션 빌드가 모두 통과했다.
- 추가 오류: 초기화본의 마이 펫/기록 화면에서 누락된 props와 보조 컴포넌트가 전체 빌드를 막았다. 기존 인터페이스에 맞춰 연결한 뒤 재빌드했다.

## 역할

나눔 목록, 나눔 글 작성, 카테고리, 사진/동영상 첨부, 좋아요, 임시저장 관련 작업을 기록한다.

- 좋아요와 임시저장은 프로필의 나의 활동과 연결되어야 한다.
