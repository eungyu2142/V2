# QNA 서비스형 피드 개편

## 요청 요약

- 기존 QNA 목록을 실제 질문 피드처럼 재구성한다.
- 검색, 상태 필터, 카테고리 필터, 정렬을 유지하면서 정보 우선순위를 제목과 본문 중심으로 바꾼다.
- 웹은 최대 2열, 모바일은 1열로 표시하고 게시글 삭제는 소유자 메뉴 안으로 이동한다.

## 분석 및 판단

- 기존 데이터는 `qnaPosts`를 Supabase에서 조회하고 상세, 댓글, 병원 첨부, 기록 첨부를 같은 화면 흐름에서 사용하고 있으므로 데이터 테이블과 조회 구조는 유지했다.
- 기존 피드 로직이 해결되지 않은 글만 표시하거나 정렬값에 따라 목록을 제외하고 있어, 상태와 카테고리를 독립적으로 필터링한 뒤 정렬하도록 변경했다.
- 댓글 수는 기존 게시글 payload와 `post_comments` 조회 결과 중 큰 값을 사용해 목록에서 최신 댓글 수를 반영하도록 했다.

## 핵심 변경 내용

- QNA 헤더에 제목과 질문 작성 버튼을 추가하고, 모바일에서는 작은 `+` 버튼으로 축약했다.
- 검색창에 실제 입력 예시 placeholder와 검색어 지우기 버튼을 추가했다.
- 상태 필터를 `전체`, `답변 대기`, `답변 있음`, `해결 완료`로 분리했다.
- 카테고리 필터를 `전체 카테고리`, `건강/증상`, `사육/관리`로 분리했다.
- 정렬을 `최신순`, `댓글 많은 순`, `오래된 순`으로 바텀시트에서 선택하도록 정리했다.
- 게시글 카드는 제목, 본문 요약, 동물 정보, 작성자/시간/댓글, 기록·사진 첨부 순서로 재배치했다.
- 카드 삭제 버튼을 점 3개 메뉴 안으로 숨기고, 본인 게시글에만 메뉴를 표시했다.
- 제목 없는 데이터 fallback, 검색/필터 결과 없음 상태, 키보드 Enter/Space 상세 이동을 추가했다.
- 기존 QNA Supabase 스키마와 상세/댓글/기록 첨부 연결은 변경하지 않았다.
