# UX 작업 기록

## 역할

사용자 흐름, 화면 전환, 검색/작성/수정/삭제/좋아요/임시저장 흐름, 기능 간 연결 경험을 기록한다.

## 기준

- 네이버 지도 UX를 많이 참고한다.
- 지도, 리뷰, 나눔, 질문, 내 펫, 캘린더, 프로필은 기록/좋아요/임시저장 데이터가 서로 연결되어야 한다.
- 병원 리뷰는 외부 사이트로 이동하지 않고 앱 내부에서 열려야 한다.

## 2026-07-03 기록

### 문제 인식

사용자는 UX 관련 판단과 완료 내용을 기능 코드와 분리해서 기록하길 원했다.

### 해결 과정

`docs/project-records/UX.md`를 만들고, 이후 화면 흐름이나 사용성 변경이 있을 때 이 파일에 누적 기록하도록 했다.

### 완료 기록

UX 작업 기록 파일을 생성했다.
