# UX 작업 기록

## 역할

사용자 흐름, 화면 전환, 검색/작성/수정/삭제/좋아요/임시저장 흐름, 기능 간 연결 경험을 기록한다.

## 기준

- 네이버 지도 UX를 많이 참고한다.
- 지도, 리뷰, 나눔, 질문, 내 펫, 캘린더, 프로필은 기록/좋아요/임시저장 데이터가 서로 연결되어야 한다.
- 병원 리뷰는 외부 사이트로 이동하지 않고 앱 내부에서 열려야 한다.

## 2026-07-03 기록

### 문제 인식

사용자는 UX 관련 판단과 완료 내용을 기능 코드와 분리해서 기록하길 원했다.

### 해결 과정

`docs/project-records/UX.md`를 만들고, 이후 화면 흐름이나 사용성 변경이 있을 때 이 파일에 누적 기록하도록 했다.

### 완료 기록

UX 작업 기록 파일을 생성했다.

## 2026-07-22 반응형 콘텐츠 탐색 흐름 개선

## 요청 요약

페이지마다 달랐던 콘텐츠 너비, 카드 크기, 정렬과 간격을 하나의 반응형 레이아웃 기준으로 통일했다.

## 분석·판단 이유

기능별 스타일이 여러 구간에서 서로 덮어쓰며 카드의 열 수와 간격이 달라지고 있었다. 기존 기능별 시각 표현은 보존하면서 최종 공통 레이아웃 레이어가 컨테이너, Grid, 카드 높이와 하단 정렬을 책임지도록 구성했다. 전체 화면 지도가 핵심인 지도 화면은 일반 콘텐츠 컨테이너 규칙에서 제외했다.

## 수정 파일

## 핵심 변경 내용

- 데스크톱 콘텐츠 최대 너비를 1280px로 제한하고 중앙 정렬했다.
- 공통 여백 토큰과 24px Grid 간격을 도입했다.
- 펫, 나눔, QNA, 프로필 목록을 데스크톱 3~4열, 태블릿 2열, 모바일 1열로 통일했다.
- 카드를 동일 너비와 높이로 늘리고 내부를 세로 Flex로 구성해 하단 액션을 수평 정렬했다.
- 입력 선택 Grid와 상세 정보 Grid도 모바일에서 안전하게 1열로 축소되도록 정리했다.
- 모바일의 툴바와 액션 영역을 전체 너비로 배치해 터치 동선을 단순화했다.
