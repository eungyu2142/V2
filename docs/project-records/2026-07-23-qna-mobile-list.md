# Q&A 모바일 목록 화면 개선

## 요청 요약

- 현재 Q&A 목록의 모바일 가독성, 정보 계층, 카드 구분, 필터/정렬, 작성 FAB 위치를 개선한다.
- 기존 Supabase 조회, 검색, 필터, 정렬, 상세 이동, 조회수, 댓글 수, 기록 첨부, 작성자 메뉴는 유지한다.

## 분석 및 판단

- Q&A 목록은 `QnaScreen`과 `QnaHelpCard`에서 렌더링되고 마지막 CSS override가 이전 스타일을 덮는 구조라, 데이터 흐름은 유지하고 최종 Q&A 전용 CSS와 카드 JSX만 조정했다.
- 모바일에서 메타 정보가 한 줄에 몰리지 않도록 작성자/시간과 조회/댓글을 두 줄로 분리했다.
- 기록 첨부는 기존 스냅샷을 사용해 유형 요약을 한 줄로 만들고 긴 구분선을 제거했다.

## 핵심 변경 내용

- 상단 제목은 `Q&A`로 유지하고 모바일 상단 여백을 24px 기준으로 축소했다.
- 검색창 높이, 테두리, 간격을 정리하고 placeholder를 `질문이나 동물 종으로 검색`으로 통일했다.
- 상태 필터는 첫 번째 행, 카테고리와 정렬은 두 번째 행으로 고정했다.
- 정렬은 `최신순`, `인기순`, `조회순`, `댓글순`을 제공하며 `정렬:` 접두어와 강한 파란색 선택 스타일을 제거했다.
- 질문 목록을 흰색 단일 열 카드로 구분하고 제목/본문/동물/메타 정보 계층을 강화했다.
- 작성자·시간과 조회수·댓글 수를 두 줄로 분리했다.
- 기록 첨부를 `기록 n개 · 유형`과 사진 개수 한 줄로 표시하고 긴 밑줄을 제거했다.
- 모바일 FAB를 하단 내비게이션 위 88px 기준으로 배치하고 목록 하단 여백을 확보했다.
- 카드 관리 버튼 aria-label을 `질문 관리 메뉴`로 통일했다.
