# profile 작업 기록

프로필, 계정 관리, 프로필 사진, 닉네임, 로그아웃, 계정 삭제, 나의 활동, 좋아요, 임시저장, 내가 쓴 글/리뷰 수정 삭제 관련 작업을 날짜별로 기록한다.

## 기준

- 좋아요, 임시저장, 내가 쓴 글/리뷰 수정 삭제는 프로필의 나의 활동과 연결되어야 한다.
- 나의 활동 항목을 누르면 해당 UI로 이동해야 한다.
- 사용자의 프롬프트로 코드를 변경할 때마다 요청 요약, 판단 이유, 수정 파일, 핵심 변경 내용, 검증 결과, 남은 작업을 한국어로 남긴다.

## 2026-07-10 프로필 임시저장 기능 연결

### 요청 요약

- 프로필에 임시저장 기능을 추가하고, 프로필을 클릭했을 때 각 기능을 실제로 이용할 수 있게 연결해 달라고 요청했다.

### 판단 이유

- 프로필 메뉴가 단순 표시가 아니라 사용자의 활동 허브가 되어야 했다.
- 임시저장은 QNA/나눔 작성 흐름과 Supabase `drafts` 테이블을 함께 연결해야 했다.

### 핵심 변경 내용

- `DraftItem` 타입을 추가했다.
- Supabase `drafts` 테이블을 `loadAppData`, `saveAppData`, `deleteAppData`로 연결했다.
- 로그인 사용자 기준으로 `drafts` 목록을 로드해 프로필 패널에 전달했다.
- QNA 작성 화면 마지막 단계에 `임시저장` 버튼을 추가했다.
- 나눔 작성 화면 마지막 단계에 `임시저장` 버튼을 추가했다.
- 임시저장 시 `draft_type`은 QNA는 `question`, 나눔은 `share_item`으로 저장했다.
- 프로필 패널에 `임시저장` 메뉴를 추가했다.
- 프로필 임시저장 목록에서 각 초안을 등록하거나 삭제할 수 있게 연결했다.
- `내가 쓴 글`, `저장한 병원`, `로그아웃`, `내 정보 수정` 메뉴도 클릭 시 패널 내부 화면이 바뀌도록 연결했다.

### 검증 결과

## 2026-07-11 프로필 임시저장 이어쓰기 수정

### 요청 요약

- 프로필의 임시저장 목록에서 초안 액션이 `등록`으로 표시되는 문제를 지적했다.
- 임시저장은 게시가 아니라 작성 화면으로 돌아가서 `이어쓰기`를 해야 한다고 요청했다.

### 판단 이유

- 임시저장의 목적은 바로 등록이 아니라 중단한 작성 흐름을 복원하는 것이다.
- 기존 `publishDraft` 흐름은 사용자의 기대와 맞지 않았다.

### 수정 파일

- `src/App.tsx`
- `docs/project-records/profile.md`

### 핵심 변경 내용

- 프로필 임시저장 목록의 액션명을 `등록`에서 `이어쓰기`로 변경했다.
- 기존 `publishDraft` 흐름을 제거하고 `continueDraft` 흐름으로 교체했다.
- 임시저장 초안을 클릭하면 프로필 패널을 닫고 QNA 또는 나눔 작성 플로우로 이동하도록 연결했다.
- QNA 작성 플로우가 draft payload의 제목, 본문, 사진, 펫, 카테고리, 연결 기록을 초기값으로 복원하도록 했다.
- 나눔 작성 플로우가 draft payload의 제목, 지역, 설명, 사진, 분류, 종, 성별 정보를 초기값으로 복원하도록 했다.
- 이어쓰기 중 다시 임시저장하면 새 초안을 만들지 않고 기존 draft id를 유지해 같은 초안을 갱신하도록 했다.
- 이어쓰기 완료 후 최종 저장하면 기존 draft를 삭제하도록 연결했다.

### 검증 결과

- `npm run build`를 실행했고 TypeScript 및 Vite 빌드가 통과했다.

### 남은 작업

- QNA와 나눔 외 작성 화면에도 임시저장을 확장해야 했다.

## 2026-07-13 프로필 사진, 내가 쓴 글, 좋아요한 목록 수정

### 요청 요약

- 프로필 사진이 URL 입력 방식인데 사진 선택 방식으로 바꿔야 한다고 지적했다.
- `내가 쓴 글`은 목록에만 있지 말고 수정/삭제가 가능해야 하며, 클릭하면 해당 글로 이동해야 한다고 요청했다.
- `저장한 병원` 표현을 `좋아요한`으로 바꾸라고 요청했다.
- 하단 탭이 너무 작으니 조금 더 크게 보이길 요청했다.

### 판단 이유

- 프로필 사진은 사용자가 파일을 직접 고르는 방식이 더 자연스럽다.
- 프로필의 나의 활동은 조회뿐 아니라 열기, 수정, 삭제까지 연결되어야 한다.
- 저장/좋아요 용어는 앱 전체에서 통일되어야 한다.

### 수정 파일

- `src/App.tsx`
- `src/App.css`
- `docs/project-records/profile.md`

### 핵심 변경 내용

- 프로필 사진 입력을 URL 입력에서 파일 선택 방식으로 변경했다.
- 선택한 사진은 Data URL로 미리보기와 프로필 저장에 사용했다.
- 프로필 메뉴의 `저장한 병원`을 `좋아요한`으로 변경했다.
- `좋아요한` 화면에 좋아요한 QNA와 나눔 글을 모아 보여주고, 열기 버튼으로 해당 글 화면으로 이동하도록 했다.
- `내가 쓴 글` 목록에 `열기`, `수정`, `삭제` 버튼을 추가했다.
- QNA 글은 프로필에서 열면 QNA 상세 화면으로 이동하게 했다.
- 나눔 글은 프로필에서 열면 나눔 상세 화면으로 이동하게 했다.
- 프로필에서 QNA/나눔 글을 수정하면 기존 글 id를 유지해 저장되도록 했다.
- QNA 수정 저장 시 기존 댓글, 좋아요, 해결 상태가 초기화되지 않도록 보존했다.
- 하단 탭의 높이, 글자 크기, 여백을 키웠다.

### 검증 결과

- `npm run build`를 실행했고 TypeScript 및 Vite 빌드가 통과했다.

### 남은 작업
