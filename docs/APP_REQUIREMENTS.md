# Special Animal Care PWA Requirements

이 문서는 사용자가 제시한 전체 기능 개요를 이 프로젝트의 기준 요구사항으로 정리한 것이다. 이후 구현, 리팩터링, UI 수정 시 이 문서를 우선 참고한다.

## 1. 메인 지도 & 리뷰

상위 UX는 네이버 지도를 많이 참고한다.

### 지도 기본 구조

- 자신의 위치를 기준으로 한 지도가 나타난다.
- 화면은 지도가 중심이고, 검색과 필터는 지도 위 오버레이로 배치한다.
- 검색창은 분류 태그 상단에 위치한다.
- 검색창 구성:
  - 병원 이름 검색
  - 내 위치 버튼
- 내 위치 버튼을 누르면 지도 중심이 내 위치로 이동한다.
- 화면 상단 분류 태그:
  - 파충류
  - 조류
  - 설치류
  - 양서류
  - 전체

### 병원 마커

- 병원은 H 마커로 표시한다.
- 기본 마커:
  - 배경은 민트색
  - H 마커는 흰색
- 좋아요한 병원은 마커 옆에 하트가 표시된다.
- 해당 분류 리뷰가 5개 이상이면 신뢰도 확인을 위해 마커 색이 파란색으로 전환된다.

### 병원 클릭 시 정보

병원 마커를 클릭하면 병원 정보가 열린다.

표시 정보:
- 병원 이름
- 내 위치로부터 떨어진 거리
- 최근 진료한 종
  - 리뷰 데이터에서 가져온다.
- 리뷰 개수
- 하트 좋아요 버튼
- 분류 태그
- 리뷰 영역

### 리뷰 화면

- 리뷰 보기는 외부 사이트로 이동하지 않고 앱 내부에서 열린다.
- 네이버 지도 리뷰 UI를 참고한다.
- 리뷰가 없으면 리뷰가 없다고 표시한다.
- 리뷰가 있으면 스크롤 가능한 목록으로 보여준다.
- 사진/영상 리뷰를 지원한다.
- 사진 보기와 글 보기 전환이 가능해야 한다.
- 리뷰는 요약되어 나열된다.
- 요약 기준:
  - 분류
  - 동물 병원
  - 종
- 상세보기에서는 요약 없이 전체 리뷰를 펼쳐서 볼 수 있다.
- 내가 쓴 리뷰에는 수정/삭제 버튼이 있다.
- 리뷰 좋아요 버튼이 있다.

### 리뷰 작성

- `+` 버튼으로 리뷰 작성이 가능하다.
- 영수증/결제 내역 인식을 지원한다.
- 리뷰 폼 필드:
  - 반려동물 선택
    - 마이 펫에 등록된 반려동물 정보가 선택지로 표시된다.
  - 진료 내용 직접 입력, 선택
  - 병명, 선택
  - 방문한 동물 병원 선택
    - 병원 리스트에서 선택 가능
  - 수의사 의견
  - 비용 입력
    - 숫자만 입력
    - 쉼표 자동 입력
    - 선택
  - 날짜 선택
    - 달력 표시
  - 처방받은 약 입력, 선택
  - 직접 작성 글
  - 사진 첨부
- 리뷰 작성 시 해당 날짜에 동물 기록이 추가된다.
- 임시 저장 기능이 필요하다.

### 리뷰 태그

기본 리뷰 태그:
- 수의사 님이 친절해요
- 병원의 위생이 좋아요
- 설명이 자세해요
- 가격이 합리적이에요
- 아이를 조심스럽게 잘 다뤄주세요
- 진료 장비가 잘 갖춰져 있어요
- 예약 시간이 잘 지켜져요
- 대기 시간이 짧아요
- 응급 상황에 빠르게 대응해요
- 검사 결과를 쉽게 알려줘요
- 집에서 관리하는 법을 알려줘요
- 동물 특성을 잘 이해해요
- 과잉진료가 없어요
- 비용 안내가 투명해요
- 병원 분위기가 차분해요
- 재방문하고 싶어요
- 태그 직접 작성 가능

## 3. 질문 QNA

- 질문 목록에서는 머리글만 나타난다.
- 클릭 후 자세한 질문이 나온다.
- 펼쳐서 더보기가 가능하다.
- 질문 작성 기능이 있다.
- 작성 폼:
  - 내 펫
  - 머리글
  - 작성 내용
  - 임시저장
- 작성 시 동물 기록에 추가된다.
- 질문 하단에는 댓글 기능이 존재한다.

  - 기타
- 입력 필드:
  - 종, 직접 입력
  - 성별
    - 수컷
    - 암컷
    - 미구분

## 5. 캘린더

- 달력은 화면 전체에 나타난다.
- 왼쪽 상단에서 내 펫 선택 가능.
- 날짜를 클릭해서 기록을 추가한다.

- 임시 저장 기능 필요.

## Cross Feature Rules

- 좋아요 데이터는 프로필의 나의 활동과 연결한다.
- 임시 저장 데이터는 프로필에서 다시 이어 작성할 수 있어야 한다.
- 리뷰 작성과 질문 작성은 동물 기록/캘린더와 연결한다.
- 지도 병원 데이터와 리뷰 데이터는 서로 연결되어야 한다.


## Hospital Data Collection Policy

- PWA frontend must not call the Naver Local Search API directly.
- Naver Search Client ID/Secret stay only in server-side environments or GitHub Actions secrets.
- Hospital base data is collected weekly from the Naver Local Search API and written to public/data/exotic-hospitals.json.
- Collection uses nationwide si/gun/gu regions with special-animal search keywords and low-concurrency sequential requests.
- Duplicate hospitals are merged by normalized hospital name and address.
- supportedAnimals stores specific matched animal categories. If no specific category is found, the app shows the hospital as 기타.
- classification is confirmed when a hospital is found through specific animal keywords, multiple queries, or has supported animals; otherwise it is candidate.
- Reviews, likes, recent treated species, and user activity remain app DB/user-data features and update immediately without waiting for weekly collection.
